## 🎤 Interview Talking Points

### "Tell me about this project"

**30-Second Version:**
"I built a credit risk model on 307,000 real loan applications from Home Credit. The key challenge was the 8% default rate - highly imbalanced. I engineered 11 features, with a weighted external credit score being most predictive. Using XGBoost, I achieved 0.758 AUC. I then applied SMOTE to improve recall to 70%, and created a 3-tier risk framework that enables automated decisions for 57% of applicants while reducing non-performing assets by 72%."

**2-Minute Version:**
"This was an end-to-end credit risk project using real data from Kaggle's Home Credit competition - 307,000 loan applications with an 8% default rate.

I started with data cleaning - the dataset had 122 features with up to 70% missing values. I kept only numeric features, dropped high-missing columns, and filled the rest with medians.

For feature engineering, I created 11 domain-specific features. The key innovation was a weighted external credit score - I gave 60% weight to EXT_SOURCE_2 because it's consistently the strongest predictor in credit risk. I also created financial ratios like credit-to-income and payment-to-credit.

I compared three models: Logistic Regression (0.735 AUC), XGBoost (0.758 AUC), and XGBoost with SMOTE (0.744 AUC). The interesting trade-off was that SMOTE reduced AUC but improved recall from 66% to 70% - for credit risk, catching more defaults is worth the slight AUC drop.

For business deployment, I created a 3-tier risk segmentation based on probability distribution: 32% low risk (auto-approve), 43% medium risk (manual review), and 25% high risk (auto-decline). This enables automated decisions for 57% of applicants while achieving 72% NPA reduction.

I used SHAP for explainability and found external credit scores, credit bureau inquiries, and goods price were the top default drivers."

### "What was your biggest challenge?"

"The biggest challenge was the severe class imbalance - only 8% defaults. Standard models would just predict 'no default' for everyone and get 92% accuracy but miss all the defaults. I addressed this three ways: using class weights in XGBoost, applying SMOTE to oversample the minority class, and tuning the decision threshold. The key insight was that SMOTE reduced AUC but improved recall - I had to explain this trade-off clearly because in credit risk, catching defaults is more important than overall accuracy."

### "What would you do differently?"

"Three things: First, I'd incorporate the categorical features - I only used numeric to start, but features like occupation and education could add signal. Second, I'd try ensemble methods like stacking XGBoost with LightGBM. Third, I'd add temporal validation - split by time rather than random split to better simulate production. Also, I'd implement monitoring for model drift since credit behavior changes over time."
